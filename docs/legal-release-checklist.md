# Legal and publication gate

The repository is prepared under Apache-2.0 but must remain private until a
qualified reviewer records each decision below. This checklist is operational
evidence, not legal advice.

- [ ] Copyright ownership and contributor assignments confirmed.
- [ ] Clean-room extraction and donor-code provenance reviewed file by file.
- [ ] Apache-2.0 patent posture approved.
- [ ] Dependency and bundled-asset obligations approved from the generated
  license inventory and SBOM.
- [ ] Project name and trademark policy approved; no privately controlled
  brand assets are present or implied to be licensed.
- [ ] Private repository, screenshots, fixtures, author identities, issue
  references, and Git history reviewed for confidential material.
- [ ] Security reporting owner and release owners accept their roles.
- [ ] Signed initial commit and signed release key ownership verified.
- [ ] External contributor completes the fixture onboarding from a fresh
  machine without private registry access.
- [ ] Repository visibility change approved as a separate recorded action.

Until every box is signed off, releases may be created only for private
readiness testing and the GitHub repository must remain private.
